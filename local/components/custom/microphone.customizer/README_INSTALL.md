# Установка компонента microphone.customizer

## Структура компонента

Компонент следует стандартной структуре Битрикс:

```
local/components/custom/microphone.customizer/
├── .description.php           - Описание компонента
├── .parameters.php           - Параметры компонента
├── class.php                  - Основная логика компонента
├── ajax.php                   - AJAX обработчик
├── README_INSTALL.md          - Инструкция по установке
├── images/
│   └── icon.gif              - Иконка компонента
├── lang/ru/
│   ├── .description.php      - Языковые файлы описания
│   ├── .parameters.php       - Языковые файлы параметров
│   └── template.php          - Языковые файлы шаблона
└── templates/.default/
    ├── template.php           - HTML шаблон
    └── assets/
        ├── css/style.css      - Стили
        ├── js/                - JavaScript файлы
        ├── image/             - Изображения (требуется копирование)
        └── mic-017.svg        - SVG микрофона
```

## Автономная работа

Компонент может работать в двух режимах:

### 1. Привязанный к товару режим
- Требует `ELEMENT_ID` и `IBLOCK_ID`
- Загружает/сохраняет конфигурацию в свойство товара
- Требует авторизации пользователя

### 2. Автономный режим
- Работает без привязки к товару (`ELEMENT_ID = 0`)
- Использует конфигурацию по умолчанию
- Не требует авторизации
- Сохранение конфигурации не поддерживается

## Установка

1. Скопируйте папку `microphone.customizer` в `local/components/custom/`
2. **ВАЖНО:** Скопируйте изображения в папку `templates/.default/assets/image/`
3. Очистите кэш Битрикс
4. Добавьте компонент на страницу:

```php
<?$APPLICATION->IncludeComponent(
    "custom:microphone.customizer",
    "",
    Array(
        "IBLOCK_ID" => 0,        // 0 для автономного режима
        "ELEMENT_ID" => 0,       // 0 для автономного режима
        "PROPERTY_CODE" => "CUSTOM_CONFIG"
    )
);?>
```

## Использование

### Автономный режим
Компонент будет работать с конфигурацией по умолчанию без привязки к базе данных.

### Привязанный режим
Укажите реальные `IBLOCK_ID` и `ELEMENT_ID` для работы с конкретным товаром.

## Особенности

- Все пути к ассетам относительные, работают внутри папки компонента
- JavaScript использует ES6 модули
- Поддерживается полноэкранный режим
- Адаптивный дизайн для мобильных устройств
- Использует Asset API Битрикса для подключения ресурсов
- Автоматическое определение путей к изображениям

## Требования к изображениям

Для корректной работы в папке `templates/.default/assets/image/` должны находиться:
